Cold context, cache misses, oversized prompts, and error loops become visible minutes after they happen, with coaching cards that explain what to change and why it matters.

Impact

Govern

Two editions, one product

Developer Edition

Frontier Cockpit Local

A complete observability stack on the developer's machine: OpenTelemetry Collector, Aspire Dashboard, Prometheus, Tempo, Loki, Grafana, scheduled jobs, and a purpose-built cockpit app at localhost:3300.

  • 100 percent local; all ports bind to 127.0.0.1
  • Content capture off by default, opt-in only
  • Runs identically on macOS, Linux, and Windows
  • AI Credits estimates, cache and context coaching
Enterprise Edition

Frontier Cockpit Hybrid

Everything in Local, plus sanitized forwarding to client-owned Azure: Container Apps collector, Application Insights, Log Analytics, Azure Monitor workspace, and Azure Managed Grafana, deployed with Bicep.

  • Raw prompts and tool payloads never leave the machine
  • Defense in depth: redaction applied locally and in Azure
  • Dev, test, and prod parameter sets; cost and data classification tags
  • GitHub Enterprise audit log and usage metrics ingestion

Metrics that change behavior

Volume metrics tell you what happened. Frontier Cockpit pairs them with efficiency and outcome metrics that tell you what to do next.

Cost and FinOps

AI Credits estimates per session, repository, and model. Monthly budget utilization with projection and alert levels. Cache savings expressed in credits, the number executives remember.

Context and cache

Hot, warm, and cold token composition. Context utilization, typical and peak. Compaction events. The coach flags cold starts and context pressure before they burn the budget.

Experience and outcomes

Time to first token, response duration, tool call depth, error rates, edit acceptance, and edit survival without revert, attributed to real Git workspaces, branches, and sessions.

Local AIU and AI Credits values are operational estimates for planning and coaching. Official GitHub Copilot billing and adoption reporting come from GitHub billing exports and the GitHub Copilot usage metrics API, which this product ingests where available.

How it works

GitHub Copilot in VS Code emits OpenTelemetry traces, metrics, and logs following the GenAI semantic conventions. Frontier Cockpit receives, stores, summarizes, and explains them.

VS Code + GitHub Copilot→ Local OTel Collector→ Tempo · Prometheus · Loki · Aspire→ Jobs: materializer + rollup→ Cockpit app + Grafana + Coach
Local Collector→ Redaction pipeline (deletes raw content)→ Azure Container Apps collector→ Application Insights · Log Analytics · Managed Grafana

Ten pinned containers, healthchecked, loopback-only, with scheduled processing inside Docker so macOS, Linux, and Windows behave identically. Validated in CI: shell quality, app builds, compose config, dashboards, secret scanning.

Privacy first, by construction

Content capture (raw prompts, responses, file paths, tool arguments and results) is disabled by default at every layer and requires an explicit opt-in per installation. In hybrid mode, dedicated pipelines delete raw content attributes before anything leaves the machine, and the cloud collector applies the same redaction again. Credentials are generated per installation; nothing ships with default passwords.

Get started

From a fresh clone to a working cockpit in three commands. Requires Docker Desktop (or Docker Engine on Linux), Git, Python 3, and VS Code with GitHub Copilot.

# macOS or Linux
git clone https://github.com/paulanunes85/frontier-cockpit.git && cd frontier-cockpit
cp local-otel/client.env.example local-otel/client.env
bash local-otel/client-bootstrap.sh

# Windows (PowerShell 7)
pwsh -ExecutionPolicy Bypass -File local-otel/client-bootstrap.ps1

Then reload VS Code, run one GitHub Copilot Chat or agent request inside a Git repository, and open http://localhost:3300.
